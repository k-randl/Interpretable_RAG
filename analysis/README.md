# Analysis Toolkit Documentation

This directory contains a comprehensive suite of scripts designed to interpret, visualize, and analyze the results of your RAG (Retrieval-Augmented Generation) pipeline. It transforms raw `.pkl` result files into actionable insights, visual plots, and browsable HTML reports.

## 🚀 Quick Start

To generate a full comparative analysis report from your results:

```bash
# Run from the project root
python3 analysis/master_analysis.py results --output_dir comparative_analysis_report
```

This will create a `comparative_analysis_report/report.html` file. Open this file in your browser to explore the results.

---

## 📊 1. Master Analysis & Reporting

### `master_analysis.py`
**The Report Generator.** This is the primary entry point for analysis. It automates the entire pipeline:
1.  **Scans** your `results/` directory for pickle files.
2.  **Identifies** experiment types (Retrieval vs. Generation) and conditions.
3.  **Generates** per-query visualizations (heatmaps, token overlap, document relevance).
4.  **Aggregates** global statistics (Position Bias, Top Tokens, POS Distribution).
5.  **Integrates** syntactic analysis from CSV logs.
6.  **Compiles** everything into a structured `report.html`.

**Key Output Features:**
*   **Global Dashboard:** Position bias plots, top influential tokens, and syntactic weight summaries at the top.
*   **Detailed Query View:** Each query gets its own row with links to a detailed `index.html`.
*   **Document Inspection:** In the detailed view, retrieved documents are hidden behind collapsible "curtains" for easy reading.
*   **Query Text:** The actual text of the query is extracted and displayed for easy tracking.

## 🧠 2. Syntax & Logic Analysis (Deep Dive)

This toolkit performs deep linguistic analysis to understand *how* your model attends to different parts of language. The results are saved in two key CSV files.

### A. `global_syntax_logic_analysis.csv`
**Generated by:** `analysis/analyze_global_syntax.py`

**Purpose:**  
Provides insights into the most "weighted" (important) syntactic patterns and functional roles across your queries. It answers: *"Does the model focus on Noun Phrases? Does it care about the Subject or Object?"*

**Content & Columns:**
*   `category`: The type of analysis (e.g., `POS_BIGRAM` (two-word patterns), `POS_TRIGRAM`, `FUNCTIONAL_ROLE`).
*   `key`: The specific pattern identified (e.g., `"NOUN + OTHER"`, `"SUBJECT"`, `"OBJECT"`).
*   `weight`: The raw accumulated importance score (attribution) for this pattern.
*   `norm_weight`: The importance score normalized by the average token weight of the query (allows cross-query comparison).
*   `text`: The actual text segment from the query that matched the pattern.
*   `type`: Indicates if this comes from a `retrieval` or `generation` task.

**Generation Process:**
1.  Iterates through all `.pkl` result files.
2.  Reconstructs full words from LLM tokens and aligns the importance scores.
3.  Uses NLP tools (NLTK/SpaCy) to tag words with Part-of-Speech (POS) tags and infer functional roles.
4.  Aggregates these patterns to find global trends.

### B. `syntax_weight_analysis.csv`
**Generated by:** `analysis/analyze_syntax_weights.py`

**Purpose:**  
A more targeted breakdown of specific grammatical structures. It isolates common "glue" patterns to see how much weight the model places on them versus the content words.

**Content & Columns:**
*   `pattern`: The specific syntactic structure (e.g., `ADP+NOUN` (Preposition+Noun), `DET+NOUN`, `ADJ+NOUN`).
*   `term1`, `term2`: The actual words forming the pattern (e.g., "in" + "throat").
*   `weight1`, `weight2`: The individual importance scores for each word.
*   `combined_weight`: The sum of the weights (`weight1 + weight2`).
*   `global_avg_weight`: The average weight of all tokens in that specific query.
*   `normalized_combined`: The combined weight divided by the global average (indicates relative importance).
*   `type`: `retrieval` or `generation`.
*   `file`: The source filename.

**Generation Process:**
1.  Extracts tokens and weights from `.pkl` files.
2.  Maps weights to NLTK-tokenized words.
3.  Scans specifically for pre-defined patterns (Preposition+Noun, Determiner+Noun, etc.).
4.  Calculates and logs the precise attribution mass falling on these structures.

## 🛠 3. Utility & Helper Scripts

### `visualize_results.py`
**Single File Inspector.**  
Generates quick heatmaps and plots for a **single** result file (`.pkl`). Use this for debugging specific queries without running the full report.
```bash
python3 analysis/visualize_results.py results/generation/path/to/file.pkl
```

### `explain_pickle.py`
**Data Peeker.**  
Prints the internal structure of a `.pkl` file (keys, tensor shapes, available metrics). Useful for understanding the raw data format.
```bash
python3 analysis/explain_pickle.py results/path/to/file.pkl
```

### Library Scripts
These scripts contain the core logic imported by the tools above:
*   **`analyze_results.py`**: Core functions to load data and calculate basic statistics (Shared by all).
*   **`analyze_retrieval.py`**: Specialized plotting logic for retrieval tasks (e.g., weighted token overlap).
*   **`generate_plots.py`**: Specialized plotting logic for generation tasks (e.g., Shapley heatmaps).
